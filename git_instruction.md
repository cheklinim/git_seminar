## Создание репозитория

Для создания *репозитория*, то есть папки, в которой будет вестись работа гита, нужно ввести в терминале команду 

    git init

## Создание коммитов

Commit - это фиксация изменения. Для создания *коммита*, нужно показать гиту, что отслеживать (добавить файл к отслежинваю версионности),  потом зафикисровать и для удобства присвоить комиту имя.

Чтобы показать, что отслеживать, нужно ввести в терминале команду 

    git add

### Добавление версионности

После того, как мы показали гиту, какой файл отслеживать, мы проводим редактирование нужной информации и по мере необходимости создаем сам коммит (фиксируем изменение и называем его для удобства).

Для непосредственного создания коммита вводим в терминале команду

    git commit -m "Название_коммита"

### Просмотр состояния

Статус Гита - это просмотр информации о том, на какой ветке мы находимся, есть ли измененная, не зафиксированная информация

Для того, чтобы узнать *статус*, вводим в терминале команду:

    git status

Важным замечанием является то, что при работе с Visual Studio Code каждое сохранение, прежде чем он станет доступно для Гита, должно быть сохранено сочетанием клавиш (для OS Windows): 

    Ctrl+S 

Для наглядности: 
* в случае, если выполнено сочетание клавиш Ctrl+S, статус Гита не отобразит никаких изменений;
* если Ctrl+S выполнено, но Гиту не указано, что он должен отслеживать, статус покажет нам, что есть изменения в неотслеживаемом файле (запись будет красная)
* если мы добавили файл к отсеживаемому. но пока не создали сам коммит - Гит покажет, какой файл изменился и что следует создать к нему коммит
* ну и про создании коммита - статус Гита покажет, что нет не зафиксированной информации, все наконец то сохрарено и мир в безопасности

### Просмотр списка (журнала) версий

Журнал версий - это журнал наших коммитов, с названиями. датами и временем создания, фио автора, уникальным хэш номером.

Чтобы показать *журнал версий*, нужно ввести в терминале команду

    git log

Чтобы показать *журнал всех версий всех веток*, нужно ввести в терминале команду:

    git log --all

Чтобы показать *журнал версий в сокращенном виде*, нужно  ввести в терминале команду:

    git log --oneline 

### Просмотр изменений 

Чтобы сравнить, чем отличаются коммиты, их можно сравнить. 

Для того, чтобы *сравнить коммиты*, вводим в терминале команду:

    git diff <хэш> <хэш>

При этом в терминале зеленым будет показано добавленное в этой версии, красным - удаленное.

### Переключение между коммитами

Между коммитами можно переключаться , при этом в рабочей зоне будет отображаться именно то содержанние, которое было зафиксированно в конкретном коммите. 

Чтобы *переключиться на коммит*, вводим в терминале команду:

    git checkout <хэш>

Важно помнить, что пока текущие изменения не будут зафиксированны, Гит не переключит рабочую зону на содержание того коммита, в который мы перешли. 

При переходе в другой коммит, в журнале он будет отмечен как Head, а коммит с последним актуальным содержанием - Master. 

# Инстрцкция по работе с Гитом, а также второе домашнее задание по теме Знакомство с контролем версий (создание веток), 16.05

## Создание ветки

Ветка создается для того, чтобы править отдельную часть документа или программы, при этом не боясь испортить основную часть. 

Для того, чтобы создать ветку, вводим в терминале команду:

    git branch <new_branch_name>

Следует помнить, что имя ветки должно быть строго на латинице. 

## Переход на новую ветку

Само создание не переносит нас автоматически на новую ветку. Чтобы начать работу именно в ней, а ни в ветке Мастер, нужно перейти на нее.

Для того, чтобы перейти на конкретную ветку, вводим в терминале команду:

    git checkout <new_branch_name>

## Альтернативный способ создания ветки

Можно совместить момент создания новой ветки и дальнейший переход в нее в одно действие (что мне кажется удобнее, точно не забудешь перейти на ветку)

Для создания ветки и одновременного перехода на нее, необходимо ввести в терминале команду:

    git checkout -b <new_branch_name>
    
## Соединение веток

После того, как ветка создана, работа в ней произведена, ее можно слить с главной веткой. 

Чтобы слить ветку с главной, необходимо перейти на ветку master и ввести в терминале команду:

    git merge <branch_name>

Важно при этом ПРОВЕРИТЬ, на какой ветке мы находимся. Нужно быть на той ветке, что является основной и которую надо дополнить. 

## Конфликт версий веток

В случае, если содержания основной ветки и содержание вливаемой ветки расходятся, мне нужно выбрать, какую версию оставить. Гит сам решить этого не может. Возможны следующие варианты:
* оставить содержание основной ветки
* согласиться на содержание вливаемой ветки
* согласиться на оба содержания и корректировать текст вручную

## Удаление слитой ветки

Когда работа с веткой закончена, для упрощения структуры коммитов и вообще для чистоты работы, слитую ветку можно удалить. 

Чтобы удалить ветку, нужно ввести в терминале команду:

    git branch -d <branch_name>

## Просмотр списка всех веток

Для просмотра списка всех веток, необходимо ввести в терминале команду:

    git branch 

## Визуализация коммитов и веток

Для наглядности удобно отобразить ветки и коммиты в виде эдакого графического дерева.

Для этого вводим в терминале команду:

    git log --all --oneline --graph

## Копирование своего репозитория в ГитХаб

Чтобы разместить свой локальный репозиторий на ГитХабе, нужно совершить следующие действия:

1. В своем аккаунте на ГитХаб создать новый репозиторий, присвоить ему имя (латинскими буквами). 
2. ГитХаб предложит три варианта создания: создать репозиторий с нуля в командной строке, скопировать существующий репозиторий из комнадной строки либо скопировать код из другого репозитория. Выбираем вариант ДВА. 
3. Следуем подсказкам ГитХаба. Копируем построчно код и вставляем в VS Code на своем локальном компьютере.

## Копирование удаленного репозитория на свой локальный компьютер

Чтобы скопировать удаленнй репозиторий на свой локальный компьютер, нужно совершить следующие действия:
1. В выбранном репозитории найти кнопку Fork (вилка) и создать этот fork. Таким образом мы копируем этот репозиторий в свои репозитории на ГитХаб, как бы делаем его своим.
2. Заходим в свои репозитории на ГитХаб. Там появвился новый, скопированный репозиторий. Находим кнопку Code. Копируем оттуда ссылку.
3. Заходим в VS Code. Вводим в терминале

    git clone <ссылка с ГитХаба>

4. Переходим в отслеживаемую папку. ДЛя этого вводим в терминале.

    cd <имя удаленного репозитория>
